About plastic.js

Data Display Framework

Data Display
Data representation in general (including tables etc.)

Framework
Not another library, but a framework to bind them all

Current Situation

  • A lot of Visualisation Libraries out there
  • Each works differently
  • Some are tied to a very specific envirionment
  • Nearly all of them require JavaScript knowledge
  • Data comes in a variety of ways
    ... which complicates everything

plastic.js Specs (1/2)

  • Open Source: github.com/Fannon/plastic.js
  • Platform neutral: Pure HTML5
  • Framework: Providing a platform for existing libraries
  • Easy to use: No programming skills required
  • Extensible: Data Modules, Query Modules, Display Modules

plastic.js Specs (2/2)

  • Modular: Loosely coupled architecture
  • Async: Runs asyncronally and possibly multithreaded
  • Dependency Management: "Smart" lazy loading
  • Schemas: Makes use of schemas, does duck-typing if that failes

plastic.js Usage

  • Include plastic.js JavaScript and CSS (as usual)
  • Write a HTML embed code
  • Data and Options are provided through HTML tags and their attributes (WebComponents Style)

Examples

Simple Table from JSON


<plastic id="table-json-file" style="height: 400px; width: 100%;">
    <script class="plastic-data" type="application/json"
            data-url="data/simple-default.json" data-format="simple-default"> </script>
    <script class="plastic-display" data-display-module="simple-table"
            type="application/json"></script>
</plastic>

Simple Bars from SPARQL Endpoint


<div class="plastic-js" style="height: 400px">
    <script class="plastic-query" type="application/sparql-query" data-query-url="http://dbpedia.org/sparql">
        PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
        PREFIX type: <http://dbpedia.org/class/yago/>
        PREFIX prop: <http://dbpedia.org/property/>

        SELECT ?country_name ?population
        WHERE {
            ?country a type:LandlockedCountries .
            ?country rdfs:label ?country_name .
            ?country prop:populationEstimate ?population .

            FILTER (?population > 12000000 && langMatches(lang(?country_name), "en")) .
        }
        ORDER BY DESC(?population)
    </script>
    <script class="plastic-options" type="application/json">
        {
            "benchmark": true
        }
    </script>
    <script class="plastic-display" data-display-module="discrete-bar-chart" type="application/json">
        {
            "staggerLabels": false,
            "tooltips": true,
            "showValues": true
        }
    </script>
</div>

Module Sequence Flow

SMW

  • CMS independend. A simple wrapper extension could integrate it.
  • Performance: Data is loaded asynchronally / multithreaded and rendered when its done. Site loading is not slowed.
  • SPARQL / ASK / CVS / JSON data (even from different sources) can share the same Visualisation modules
  • SMW Data can be queried / visualized on ANY page

Current State

In early development

  • Working Architecture
  • Several data formats supported
  • Able to query SPARQL Endpoints and semantic Wikis
  • A few simple display modules (tables, bars, pies)
